Load a Specific Reminder

Today represents reminders using its internal Reminder model. Before writing reminder data to the event store, you need to retrieve the corresponding EKReminder object. In this section, write a new method that retrieves an EKReminder object for a given identifier.

Шаг 1

Add a method named readReminder to the bottom of the extension in ReminderListDataSource.swift.


The method accepts a string and a completion handler of type (EKReminder?) -> Void.

Start by adding a method to fetch an individual reminder from the event store.

Шаг 2

Call the completion handler with nil and exit the method if the user hasn’t granted access to their reminders.

Шаг 3

Query the event store for an item with an identifier that matches the id parameter.


The calendarItem(withIdentifier:) method returns an optional EKCalendarItem, which is the superclass for reminders and calendar events. You need to unwrap the optional and cast it as an EKReminder.

Items in the event store have a unique calendarItemIdentifier property. To read a reminder from the store, you search the event store for a calendar item with a calendarItemIdentifier that matches the id passed to the function. The calendarItem(withIdentifier:) method returns the first occurrence of an event or reminder with the specified identifier.
